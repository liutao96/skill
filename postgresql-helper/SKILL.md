# PostgreSQL Helper

结合 PostgreSQL 官方文档为用户提供专业的 SQL 问题解决方案。

## 核心工作流程

遵循以下 5 步诊断流程：

### 1. 理解问题

快速识别问题类型：
- **错误诊断**：用户提供错误信息或报错代码
- **功能查询**：询问特定函数、语法、数据类型
- **性能优化**：查询慢、需要索引建议
- **查询编写**：需要帮助编写复杂 SQL

### 2. 定位文档章节

根据问题类型，访问对应的文档章节（参考 [doc-index.md](references/doc-index.md) 获取章节 URL）：

**常见映射：**
- 语法错误 → 第 4 章 SQL Syntax
- 函数用法 → 第 9 章 Functions and Operators
- 数据类型 → 第 8 章 Data Types
- 性能问题 → 第 11 章 Indexes + 第 14 章 Performance Tips
- 查询语句 → 第 7 章 Queries
- 错误代码 → 附录 A Error Codes

### 3. 查询官方文档

使用 WebFetch 工具访问具体章节：

```
WebFetch(
  url: "https://www.postgresql.org/docs/current/[章节路径]",
  prompt: "详细总结 [具体内容]，包括语法、参数、示例和注意事项"
)
```

**查询策略：**
- 优先查询最相关的单个章节
- 如需对比多个选项，可并行查询多个章节
- 对于复杂问题，可分步查询（先主章节，再子章节）

### 4. 分析和提取

从文档内容中提取：
- ✅ 问题的根本原因
- ✅ 官方推荐的解决方案
- ✅ 完整的语法说明
- ✅ 可直接使用的代码示例
- ✅ 相关的注意事项和最佳实践

### 5. 提供解决方案

按照以下格式输出：

````
## 问题分析
[简要说明问题原因]

## 解决方案
[官方推荐的方法]

```sql
-- 完整的代码示例
[可直接运行的 SQL 代码]
```

## 说明
[参数解释、使用场景、注意事项]

## 最佳实践
- [建议 1]
- [建议 2]

## 相关资源
- [章节名称](文档链接)
````

## 输出要求

### 代码优先
- 提供完整、可运行的 SQL 代码
- 代码注释清晰标注关键部分
- 包含实际数据示例

### 清晰解释
- 使用中文解释技术概念
- 技术术语保持英文（如 `INNER JOIN`, `WHERE`）
- 突出重点，避免冗长

### 准确引用
- 始终在结尾提供文档来源链接
- 格式：`来源：[章节标题](https://www.postgresql.org/docs/current/...)`

## 常见问题类型处理

### 错误诊断
1. 识别错误代码（如 `ERROR: 42P01`）
2. 查询附录 A（Error Codes）或相关章节
3. 说明错误原因 + 提供修复代码

### 函数查询
1. 访问第 9 章对应小节
2. 提取函数签名、参数说明
3. 提供 2-3 个实用示例

### 性能优化
1. 分析查询结构
2. 查询索引章节和性能建议
3. 提供 EXPLAIN 分析 + 索引建议 + 优化后的查询

### 数据类型选择
1. 访问第 8 章对应类型
2. 对比不同类型的特点（存储、性能、功能）
3. 给出选择建议和迁移方案
