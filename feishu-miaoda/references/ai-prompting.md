# AI 对话技巧与提示词编写

## 什么是 Vibe Coding

Vibe Coding 是用自然语言描述需求，让 AI 生成代码的编程方式。不需要编程基础，核心能力是**清晰表达需求**。

适用场景：快速验证产品想法（MVP）、个人工具和内部系统、没有技术背景的产品经理/设计师/创业者。

## 提示词黄金结构

```
角色设定 + 任务描述 + 背景信息 + 约束条件 + 输出格式
```

## 五大核心技巧

1. **明确角色**：告诉 AI 它是谁（如"你是一个专业的前端工程师"）
2. **描述具体场景**：给出具体的使用场景和目标用户，不要泛泛而谈
3. **提供参考范例**：上传截图、贴入示例代码，让 AI 有参照物
4. **设定约束条件**：明确不需要什么（如"不要使用第三方库"、"风格要简洁"）
5. **分步骤迭代**：复杂需求拆分成多个步骤，逐步完善

## 常见错误 vs 正确写法

| 错误写法 | 改进写法 |
|----------|----------|
| "做一个网站" | "做一个团队活动报名页面，包含活动介绍、日程安排、报名表单" |
| "改一下样式" | "把背景色改成深紫色 #2D1B69，字体改成白色，卡片加 8px 圆角" |
| "加个功能" | "在页面底部加一个悬浮按钮，点击后平滑滚动回顶部" |
| "有问题修一下" | "现在点击提交按钮后页面空白，我希望提交成功后显示'报名成功'提示" |

## 问题定位法

出现问题时，使用以下格式描述：
> "现在的效果是 X，我希望的效果是 Y，请帮我修改"

## 进阶策略

- **参考图优先**：先去花瓣网、Dribbble 找参考截图 → 发给 AI 分析结构 → 生成提示词
- **让 AI 改进提示词**：先写粗糙版本，让 AI 帮你优化后再执行
- **用 GPT/Gemini 生成提示词**，再把提示词粘贴给妙搭执行（提示词生成与执行分离）
- **批量下达细节命令**：一次性输入多个细节修改要求，妙搭会依次执行，无需等待

## 整体 vs 细节的调整策略

| 调整类型 | 推荐方式 |
|----------|----------|
| 整体风格/排版调整 | 截图 + 期望描述 → AI 生成提示词 → 妙搭全局执行 |
| 单个元素（字色/图片/间距） | 妙搭编辑器直接选中该元素 → 下达精确修改命令 |

## 完整开发流程

```
明确需求（3个核心功能以内）
  → 编写提示词（参考上述结构）
  → 生成初版（先跑通主流程，不追求完美）
  → 测试反馈（描述问题 + 期望效果）
  → 迭代优化（每次只改一处，便于排查）
  → 发布上线
```

## 注意事项

- 第一版不追求完美，先验证核心功能可用
- 每次只修改一个地方，出问题容易定位
- 复杂功能先用简单数据测试，验证逻辑后再接真实数据
- 保存好可用版本，避免改坏后找不回来
