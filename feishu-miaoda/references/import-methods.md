# 导入方式

导入入口：https://miaoda.feishu.cn/import

## 从多维表格导入

### 适用场景

已有飞书多维表格（Bitable）的表结构和数据，希望快速构建以其为数据源的管理应用。

### 操作步骤

1. 进入导入页面，选择「从多维表格导入」
2. 搜索或选择目标多维表格文档
3. 选择要导入的具体数据表，预览字段结构
4. 可选：勾选「导入原表数据」以同步历史数据
5. 点击「开始开发」进入应用开发界面

### 导入内容说明

| 会被导入 | 暂不支持导入 |
|----------|--------------|
| 数据表表结构（字段定义） | 附件字段 |
| 数据表中的数据（可选） | 群组字段 |
| | 查找引用字段 |
| | 地理位置字段 |
| | 关联字段 |
| | 外租户人员数据 |

**注**：人员字段包含外租户人员时，外部人员值被忽略，不阻塞当前行导入。

### 注意事项

- 导入后原多维表格数据不受影响
- 当前为一次性导入，不持续自动同步
- 若需双向同步，需等待后续版本支持

## 从 ZIP 文件导入

### 适用场景

已有本地 React 前端项目，需要迁移到妙搭平台继续开发和托管。

### 文件要求

- 包含完整项目文件（源代码、配置文件、package.json 等）
- 文件大小不超过 **50MB**
- 支持框架：**React**（配合 Tailwind CSS）
- 支持语言：JavaScript、TypeScript

### 推荐 ZIP 目录结构

```
my-project.zip
└── my-project/
    ├── package.json
    ├── src/
    └── public/
```

### 操作步骤

1. 登录妙搭，进入导入页面
2. 选择「上传 .zip 文件」
3. 点击上传区域，选择本地 ZIP 文件
4. 等待妙搭助手完成项目校验
5. 进入开发界面，开始迁移和二次开发

### 导入内容对照

| 会被导入 | 不会被导入 |
|----------|------------|
| 应用代码（前端代码和逻辑） | 环境变量（.env 文件）|
| UI 组件（界面组件、样式） | node_modules |
| 静态资源（图片、图标） | 构建产物（dist/build）|
| 项目配置（package.json 等） | 平台特定配置文件 |

### 常见问题排查

| 问题 | 解决方案 |
|------|----------|
| ZIP 上传失败 | 检查文件大小（≤50MB）、文件完整性、网络稳定性 |
| "无法识别的项目类型" | 确保根目录包含 package.json |
| 运行时报错 | 检查环境变量配置、依赖版本兼容性；或向妙搭助手描述错误让其诊断 |

**注意**：不要将 node_modules 打包进 ZIP，会增大体积且引发兼容问题。GitHub 仓库直接下载的 ZIP 可直接导入。

## 从其他平台导入

| 平台 | 说明 |
|------|------|
| Replit | 从 Replit 项目直接导入 |
| Lovable | 从 Lovable 生成的项目导入 |
| Google AI Studio | 从 Google AI Studio 导入 |
