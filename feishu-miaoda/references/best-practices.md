# 实战经验与最佳实践

## 通用开发原则

- **核心功能优先**：第一版只做 3 个以内的核心功能，先跑通主流程
- **每次只改一处**：修改后立即测试，出问题容易定位根因
- **保留可用版本**：改之前备份，避免改坏后找不回来
- **简单数据先测**：复杂功能先用 mock 数据验证逻辑，再接真实数据

## 活动页面搭建（视觉导向）

**适用场景**：团建、年会、产品发布会等活动宣传页，功能简单但视觉要求高。

### 五步工作流

```
提示词制作 → 初版网页生成 → 视觉图片制作 → 样式精调 → 发布
```

**步骤一：提示词制作**
- 先去花瓣网、Dribbble 找参考截图
- 将截图发给 GPT/Gemini，让它分析结构后生成高质量提示词
- 明确每个区块内容（Banner、日程区、报名区等）

**步骤二：生成初版**
- 将提示词粘贴到妙搭对话框
- 目标是跑通基本结构，不追求视觉完美

**步骤三：制作配图**
- 工具推荐：即梦、Lovart、Nano Banana 等 AI 生图工具
- 先找风格参考图 → 发给生图工具 → 生成风格一致的配图
- 推荐风格：3D 立体卡通风（视觉冲击力强，适合活动页）

**步骤四：样式精调**
- **整体风格**：截图 + 期望描述 → GPT 生成修改提示词 → 妙搭全局执行
- **细节元素**：在妙搭内直接选中元素（图片/文字/间距）→ 下达精确修改命令
- **批量命令**：一次性下达多个细节修改，妙搭依次执行，效率更高

## 数据管理应用搭建（排班/CRM/管理系统）

**适用场景**：智能排班、客户管理、库存管理等数据驱动的管理系统。

### 关键步骤

1. **梳理规则优先**：将所有业务规则逐条写清楚，一次性输入给 AI，而非分多次描述
2. **数据表设计**：用妙搭内置数据库存储人员信息、规则配置、历史记录
3. **提供样例数据**：给 AI 一份手工处理过的样例，让它理解期望格式
4. **小范围先测**：先用 5 条以内数据测试规则逻辑，验证无误再扩展到全量

### 排班系统规则配置示例

```
班次配置：早班 08:00-16:00，晚班 16:00-00:00
人员约束：标记"特殊"的员工不排夜班
连班限制：同一人不能连续超过 5 天
休息保障：每周最少休息 2 天
公平性：周末班次在团队内轮流分配
技能匹配：岗位A只能由有"A证"标签的人员担任
优先级：优先排入已批准的休假申请，再做整体排布
```

## 打包成桌面/手机 App

妙搭应用本质是网页，可以通过第三方工具套壳成独立 App。

### 桌面 App 打包

**方案一：PakePlus（新手推荐）**
- 有图形界面，操作简单，无需命令行
- 下载安装 PakePlus → 填入妙搭应用网址 → 设置应用名称 → 点击「开始打包」
- 约 30 秒生成：Mac 生成 `.dmg`，Windows 生成 `.exe`
- Mac 首次打开提示"应用已损坏"：系统设置 → 隐私与安全性 → 点击「仍要打开」

**方案二：Pake 命令行（进阶）**
```bash
# 安装
npm install -g pake-cli

# 基本打包
pake https://你的应用地址.com --name "应用名称"

# 完整示例（含图标和窗口大小）
pake https://xxx.com --name "我的应用" --icon ./logo.png --width 1000 --height 700
```

### 手机 App 打包

**方案一：添加到主屏幕（最简单）**
- iPhone：Safari 打开 → 底部「共享」→「添加到主屏幕」
- Android：Chrome 打开 → 右上角菜单 → 「添加到主屏幕」
- 局限：仅供个人使用，无法分发

**方案二：PakePlus 打包**
- 选目标平台 Android → 生成 `.apk`，可直接发给他人安装
- iOS 生成 `.ipa` → 需开发者账号签名才能分发（较麻烦）

### 注意

- 打包后仍需联网使用（本质是网页套壳）
- 妙搭内容更新后，App 打开时自动加载最新版本

## 常见问题排查

| 问题现象 | 排查方向 |
|----------|----------|
| 提交按钮无反应 | 检查事件绑定是否正确；查看运行日志报错 |
| 数据不显示 | 检查数据绑定路径；SQL 查询是否有结果 |
| 样式不符合预期 | 选中元素后用精确命令描述期望效果 |
| 自动化任务不触发 | 检查任务是否已启用；查看任务日志 |
| 发布后页面空白 | 检查构建日志；确认是否有未处理的 JS 错误 |
| 飞书登录失败 | 确认回调 URL 配置正确；检查管理员是否授权 |
