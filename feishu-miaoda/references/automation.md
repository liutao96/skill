# 自动化任务

## 概述

在妙搭应用中配置定时或触发式的自动化工作流，实现数据处理、通知推送、跨系统联动等场景，无需人工干预。

## 支持的触发器类型

| 触发器 | 说明 | 配置示例 |
|--------|------|----------|
| 定时触发 | 按 Cron 表达式周期执行 | `0 9 * * 1-5`（工作日早9点） |
| 数据库变更触发 | 数据插入/更新/删除时触发 | 新订单插入时自动发通知 |
| HTTP 触发（Webhook） | 通过 Webhook URL 触发 | 外部系统调用时触发 |
| 手动触发 | 在管理界面手动执行 | 测试和调试时使用 |

## 支持的动作类型

| 动作 | 说明 |
|------|------|
| 数据库操作 | SELECT / INSERT / UPDATE / DELETE |
| 发送飞书消息 | 文本消息、卡片消息，发给指定人或群组 |
| 调用 HTTP API | GET / POST 等，调用外部服务 |
| 执行自定义代码 | JavaScript 逻辑处理 |
| 文件存储操作 | 读写妙搭内置文件存储 |

## 操作步骤

1. 进入「应用设置 → 自动化任务」模块
2. 点击「新建任务」
3. 选择触发器类型并配置参数
4. 添加执行动作（可多个，串行执行）
5. 点击「测试运行」验证逻辑
6. 确认无误后「启用任务」

## 典型使用场景

**场景一：定时数据汇报**
```
触发器：每天 18:00（Cron: 0 18 * * *）
动作1：SELECT 查询当日数据统计
动作2：发送飞书消息，将统计结果发给指定群组
```

**场景二：新数据自动通知**
```
触发器：数据库 orders 表插入新记录时
动作1：查询新订单详情
动作2：向相关负责人发送飞书卡片消息
```

**场景三：外部系统联动（Webhook）**
```
触发器：HTTP POST 请求到 Webhook URL
动作1：解析请求数据
动作2：写入妙搭数据库
动作3：触发后续业务流程
```

## 注意事项

- 定时任务最小间隔根据套餐不同有所差异
- 任务执行日志可在「日志 → 任务日志」中查看
- 支持失败重试配置
- 多个动作串行执行，前一个动作失败会中断后续动作
